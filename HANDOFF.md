## 当前页面 / 视图梳理

本项目是单页静态应用，入口为 `index.html`，通过 `script.js` 中的 `state.currentView` 与 URL hash 切换视图。主要路由和页面如下。

### 1. 创作中心首页

- 路由：`#/home`，空 hash 或 `#/` 也会进入该页。
- 页面名称：`创作中心`
- 入口：顶部导航 `创作中心`
- 主要功能：
  - 展示三种创作入口：`工作流`、`全自动 AI 生成`、`无限画布`。
  - `工作流` 和 `无限画布` 的开始按钮会打开创建项目弹窗；`全自动 AI 生成` 当前禁用，展示为即将开放。
  - 展示 `平台作品展示` 视频列表。
  - 视频列表支持 `最新` / `最热` 排序。
  - 只展示状态为 `已发布` 且未下架的视频。
  - 点击视频卡片打开视频预览弹窗。
  - 首页底部提示项目会自动保存至个人库。

### 2. 项目管理 / 我的项目

- 路由：`#/projects`
- 页面名称：`我的项目`
- 入口：顶部导航 `项目管理`
- 主要功能：
  - 展示项目卡片列表。
  - 支持搜索项目名称。
  - 支持按时间正序、时间倒序、名称正序、名称倒序排序。
  - 支持创建新项目。
  - 支持批量操作、全选、批量删除。
  - 支持单个项目上传封面、编辑项目名称、删除项目。
  - 点击项目卡片打开 `创作模式` 弹窗。
  - 创建项目后也会打开 `创作模式` 弹窗。

### 3. 工作流首页

- 路由：`#/project/:projectId/workflow`
- 页面名称：`工作流首页`
- 入口：项目卡片打开 `创作模式` 弹窗后，选择 `工作流` 进入。
- 主要功能：
  - 作为项目进入工作流模式后的首页 / 主控制台。
  - 左侧展示当前项目名称、画面比例、风格，以及当前模式 `工作流`。
  - 左侧工作流菜单包含：主控制台、我的项目、资产库、剧本拆解、拍摄台、视频工坊、试验台。
  - 主内容区展示项目名和风格标签。
  - 展示四个流程步骤：剧本构建、素材草图、融合草图、视频生成准备。
  - 展示脚本生成记录计数，当前为空状态。
  - 底部展示后端连接状态。
  - `构建历史` 当前为入口展示。
  - 左侧 `切换模式` 会打开创作模式弹窗，可切换到无限画布。

### 4. 项目画布列表 / 无限画布首页

- 路由：`#/project/:projectId/canvases`
- 页面名称：`{项目名} · 画布列表`
- 入口：项目卡片打开 `创作模式` 弹窗后，选择 `无限画布` 进入；也可从画布详情返回。
- 主要功能：
  - 展示当前项目下的画布列表。
  - 作为项目进入无限画布模式后的首页。
  - 左侧展示当前项目名称、画面比例、风格，以及当前模式 `无限画布`。
  - 支持搜索画布名称。
  - 支持排序。
  - 支持新建画布。
  - 支持批量选择和批量删除画布。
  - 支持单个画布上传封面、重命名、删除。
  - 点击画布卡片进入画布详情页。
  - 左侧 `切换模式` 会打开创作模式弹窗。

### 5. 无限画布详情

- 路由：`#/project/:projectId/canvas/:canvasId`
- 页面名称：当前画布名称
- 入口：项目画布列表中点击画布卡片。
- 主要功能：
  - 展示可拖拽、缩放、平移的画布编辑器。
  - 支持空画布状态。
  - 支持添加节点面板，添加文本、图片、视频、音频、上传资源等节点入口。
  - 支持双击画布创建节点。
  - 支持图片节点、文本节点、720 全景节点、全景截图组节点。
  - 支持节点选择、拖拽、删除、复制、粘贴、创建副本。
  - 支持节点左右添加相邻节点，并自动连线。
  - 图片节点工具栏包含反推、全景、多角度、打光、九宫格、擦除、洗图、宫格切分等入口，其中反推和洗图有扣星钻确认弹窗。
  - 支持图片节点上传替换图片。
  - 支持摄像机控制弹层，可设置相机、镜头、焦距、光圈，并开关摄像机控制。
  - 支持由图片节点生成 720 全景节点。
  - 全景节点支持当前视角截图、4 大视角截图、12 大视角截图、重置视角、构线参考图、全屏预览。
  - 支持右键节点打开上下文菜单：保存到资产库、创建副本、复制、粘贴、删除。
  - 画布名称右侧支持分享画布，弹窗展示分享链接、一键复制入口、访问权限开关（公开访问 / 仅自己可见）。
  - 点击分享弹窗中的 `一键复制` 会进入分享查看页。
  - 支持底部缩放控制与百分比重置。

### 6. 画布分享查看页

## 部署方法

本项目线上地址：

- `https://phantymovie-canvas-demo.vercel.app`

部署方式：

- Vercel 已连接 GitHub 仓库。
- 推送到 `origin/main` 后，Vercel 会自动从 GitHub `main` 分支部署。
- 通常等待 1-2 分钟后刷新线上地址即可看到更新。

本项目当前没有 `.openai/hosting.json`，也没有本地构建步骤；页面是静态文件，部署依赖 GitHub push 触发 Vercel 自动部署。

## 注意事项

- 不要提交：
  - `.DS_Store`
  - `assets/.DS_Store`
- 项目是纯静态原型，没有后端接口。
- 视频上传使用浏览器本地 `FileReader` / `URL.createObjectURL` 模拟。
- 审核、上下架、发布状态都保存在浏览器 `localStorage` 中。
- 如果需要重置演示数据，可在浏览器中清理站点 localStorage。
